# Operations runbooks

These documents are for use when something goes wrong in production.
Read them in advance so you know they exist when you need them.

- **[incident-response.md](./incident-response.md)** — what to do when something breaks
- **[db-restore.md](./db-restore.md)** — how to restore from a Supabase backup
- **incidents/** — log of past incidents (start empty)

When adding a new runbook, add it to this index and link from
the relevant section of `incident-response.md`.
